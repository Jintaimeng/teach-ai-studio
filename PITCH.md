# 超级教师工作台 · 路演说明

> **一句话**：面向考研/升学辅导老师的 AI 工作台，把「择校 → 调剂 → 直播答疑 → 招生推广」全链路工作，用一个对话式产品打通——**所有结论都来自真实录取数据，而非模型臆测**。

---

## 1. 我们解决什么问题

考研辅导老师每天在做的几件事，今天是割裂且低效的：

| 场景 | 现状痛点 | 本产品方案 |
|------|---------|-----------|
| **给学生择校** | 翻 Excel、查院校官网、凭经验拍脑袋 | 对话式生成「冲/稳/保」分层择校报告 |
| **指导调剂** | 信息差大、政策乱、时间紧 | 一键生成调剂/征集志愿方案 |
| **直播答疑** | 被学生问到具体分数线时现查现卡壳 | **「言出法随」**：说话即检索，实时弹出录取数据 |
| **做招生推广** | 手写宣传文案、追不上行业资讯 | 基于研bot资讯一键生成宣传文案 |

**核心差异点**：市面上的 AI 工具大多让大模型「编」答案。我们把大模型只用在**理解意图**和**组织语言**上，**录取分数、院校数据全部来自 yanbot 考研开放接口的真实历年录取数据**——可信、可溯源。

## 2. 产品全景（一张图看懂）

```
┌─────────────────────── 超级教师工作台 ───────────────────────┐
│                                                              │
│  概览        工作台首页（各模块入口）                          │

│              调剂报告  ──┤ 对话式 + 真实录取数据 + 冲/稳/保分层  │
│                                                              │
│  教学        AI 助手对话    备课/答疑/出题                     │
│              志愿推荐        按考生信息智能推荐院校              │
│              言出法随        直播按住说话，实时检索录取数据      │
│              案例收藏库      沉淀优质推荐案例                   │
```

模块由**单一注册表**（`src/config/modules.ts`）驱动——菜单、路由、首页入口卡片全部自动生成。**新增一个 AI 功能模块只需加一条配置，无需改外壳代码**，这是产品能快速扩展的工程底座。

## 3. 核心功能亮点

### 🎓 择校 / 调剂报告 —— 对话式，数据驱动
- 老师用自然语言描述考生（初试分数、意向专业、地区、院校层次），AI 逐步追问补全信息。
- 后台调用 yanbot 接口拉取真实「一志愿录取最低分」，按分差自动分**冲（-30~-8分）/ 稳（-7~+8分）/ 保（+9~+40分）** 三档。
- 输出带推荐理由的报告卡片：录取分差、985/211/双一流标签、招生院系、地理位置，并附备考提示。
- **纯数据计算，无 LLM 幻觉**——分数一定是真的。

### 🎯 志愿推荐 —— 秒回 + 智能解析双模式
- **智能档**：大模型解析「我学生 350 分，想报北京的计算机专硕」这类自然语言 → 结构化查询。
- **快档**：本地正则即时解析，追求秒回（为直播场景设计）。
- 快捷按钮重查（放宽分数/调整层次）**不走大模型**，直接合并筛选增量，省时省钱。

### 🎤 言出法随（直播答疑神器）—— 最具差异化的功能
- 直播中**按住空格说话**，对接**火山引擎大模型流式语音识别**（双向流式 WebSocket）。
- 边说边把识别文本喂给「快档」推荐引擎，**话音未落，相关院校录取数据已弹在屏幕上**。
- 解决老师直播被问到具体分数线时「现查现卡壳」的尴尬。

### 📢 推广神器 —— 资讯到文案的自动化
- 通过 MCP 接入「研bot」行业资讯/数据。
- 勾选几条资讯，大模型 SSE 流式生成宣传文案，可保存复用。

### 💬 AI 助手 + 📚 案例库
- 通用教学助手：备课、改写课文、生成练习题。
- 案例收藏库：把优质推荐结果沉淀下来，可二次生成推广文案。

## 4. 技术架构（为什么稳）

```
   浏览器 (React 18 + TDesign + Tailwind)
        │  SSE 流式 / REST
        ▼
   Express 后端 (TypeScript, ESM)
        ├── CodeBuddy Agent SDK ──── 大模型对话 / 工具调用 / 权限控制
        ├── yanbot 开放接口 ──────── 真实考研录取数据（HMAC-SHA256 鉴权）
        ├── 火山引擎 ASR ─────────── 实时语音识别（流式 WebSocket）
        ├── MCP Server（研bot）───── 行业资讯接入
        └── sql.js (WASM SQLite) ── 会话/消息/案例/文案持久化
```

**关键技术点：**
- **流式体验**：对话、报告、文案全部用 **Server-Sent Events** 实时输出，无需等待整段生成。
- **数据可信**：录取数据走 yanbot 开放接口，签名鉴权，与官方后端一致。
- **可扩展架构**：模块注册表 + Agent 预设，新功能即插即用。
- **权限可控**：SDK 提供 `default / acceptEdits / plan / bypassPermissions` 四档权限模式，工具调用可视化、可拦截。
- **零运维存储**：sql.js 纯内存 + 落盘，无需独立数据库进程。

**技术栈一览**：Node.js · Express · TypeScript · React 18 · Vite · TDesign · Tailwind · CodeBuddy Agent SDK · sql.js · 火山引擎 ASR

## 5. 数据可信链（差异化护城河）

> 这是我们和「纯大模型问答工具」最大的区别。

```
老师自然语言  ──►  大模型【只负责理解意图】  ──►  结构化查询
                                                    │
                                                    ▼
                                     yanbot 真实历年录取数据
                                                    │
                                                    ▼
                              规则引擎分层（冲/稳/保）+ 大模型【只负责组织语言】
                                                    │
                                                    ▼
                                        可溯源的报告 / 推荐卡片
```

大模型**永远不编造分数**——它只做它擅长的（理解和表达），数字交给真实数据。

## 6. 演示动线（路演建议）

1. **首页** → 展示工作台全景，强调「一个产品打通全链路」。
2. **择校报告** → 输入「我学生 360 分想报计算机」，看 AI 追问 → 秒出冲/稳/保报告，**点开理由强调分数来自真实数据**。
3. **言出法随**（高光时刻）→ 现场按住说话，话没说完数据已弹出，制造「Wow」。
4. **推广神器** → 勾选资讯，流式生成一条宣传文案，展示商业闭环（招生）。
5. **收尾** → 一句话回到价值：**「让每个辅导老师都有一支数据团队 + 一个文案团队 + 一个直播助手。」**
